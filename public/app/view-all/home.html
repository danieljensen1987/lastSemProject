This application shows our idea of how to create a study points system for teachers and students.
The original seed has been severely modified and can now handle "real" authentication
and use roles to distinguish between what logged on users can do.

  • Home Can be seen by all
  • My Profile Access to data (REST) requires user to be logged on as a Student
  • MyClasses, add and Edit Access to data (REST) requires user to be logged on as Teacher or Admin

Using the Security Features in this seed - Server:

All REST services placed in the file REST_Admin_API.js requires admin rights

All REST services placed in the file REST_User_API.js requires user rights

Using the Security Features in this seed - Client:

A token with credentials is automatically attached to all outgoing AJAX requests (when you are logged on)
